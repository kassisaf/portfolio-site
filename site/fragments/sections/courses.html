CST 300 - Major ProSeminar

View catalog description (external)

What I learned:

In CST 300 I conducted research into the emerging Virtual Reality (VR) industry shortly after Facebook began rebranding the Oculus Quest 2. I learned a lot about the history of VR and key players in the industry today, as well as how to formulate career goals through a combination of research and introspection.

I also wrote a paper examining the ethics around controversial surveillance tech being used in classrooms. With recent advancements in generative AI, the skills to examine and scrutinize how we design and deploy the technologies we create are critically important.

Major projects:

  • Industry Analysis research paper
  • Ethical Argument research paper
  • Group video presentation on nanotechnology

CST 363 - Introduction to Database Systems

View catalog description (external)

What I learned:

CST 363 was one of my favorite classes at CSUMB, and certainly one of the most practical. I learned all about database architecture, Entity Relationship (ER) design, data normalization, MapReduce algorithms, and of course, SQL.

As a result of this class, I am now fully comfortable designing relational database schemas and writing and optimizing complex SQL queries. I have also gained a reasonable working knowledge of document databases such as MongoDB.

Major projects:

  • Database design for a fictional pharmacy chain

CST 311 - Introduction to Computer Networks

View catalog description (external)

What I learned:

In CST 311, I learned about each of the layers in the OSI model, UDP and TCP/IP, routing algorithms, network architecture, and cryptography. I used Mininet with Python to simulate and experiment with various network topologies, and used Wireshark to dissect packets sent between nodes.

Major projects:

  • Encrypted chat program with client and server apps

CST 336 - Internet Programming

View catalog description (external)

What I learned:

CST 336 was a heavily project-oriented course with a high degree of creative freedom, in which I built many websites of increasing complexity each week. While I started the course with existing HTML and CSS knowledge, I learned a lot about JavaScript, Node.js, Express, and the EJS templating engine.

First I learned how to use JavaScript to manipulate the DOM and consume external APIs via fetch calls, and later I used Node.js and Express to construct my own REST APIs. As a culminating project, I worked with a team to design and build a practical, full-stack website designed to help job-seekers organize job applications throughout their search.

Major projects:

  • Pokemon API
  • AppliTrack (job application tracker website)

CST 438 - Software Engineering

View catalog description (external)

What I learned:

In CST 438, I learned about the Software Development Lifecycle (SDLC), development methodologies including Agile and Waterfall, best practices for testing, and how to develop and document software requirement specifications (SRS)

I built a microservice for a course management application in Java SpringBoot with a React frontend, and later integrated it with a gradebook microservice developed in parallel by another student. This required significant coordination to ensure that our services could communicate through a common message broker (RabbitMQ). We utilized Mockito for unit testing, Selenium for end-to-end UI testing, and Postman for API testing, as well as Pivotal Tracker for managing user stories and storing design prototypes.

Major projects:

  • Class Registration Microservice
